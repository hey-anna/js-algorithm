## ✅ 입국심사 - 이진 탐색 포인터 이동 시각화

### 📥 입력

```js
n = 6;
times = [7, 10];
```

### 🧠 초기 상태

```
left = 1
right = 60   ← (10 × 6) → 가장 느린 심사관이 전부 심사할 경우
```

---

### 🔄 이진 탐색 진행 흐름 (mid 계산 & 포인터 이동)

```text
1     15    22    26    27   28   29     30                             60
|─────|─────|─────|─────|────|────|──────|──── mid ↑ right ↓ (7명 가능)
 ↑                                     ↑
left                                  right
         ↓ mid = 15 (3명) → 부족 → left = 16
         ↓ mid = 22 (5명) → 부족 → left = 23
         ↓ mid = 26 (5명) → 부족 → left = 27
         ↓ mid = 28 (6명) → 가능 → answer = 28, right = 27
         ↓ mid = 27 (5명) → 부족 → left = 28
         → loop 종료 (left > right)
```

---

### 🏁 최종 도달

```
left = 28
right = 27 → 종료 조건 (left > right)

🎯 answer = 28분
```

---

### 💬 요약

- `mid`는 `(left + right) / 2` 로 계속 갱신됨
- 처리 가능 인원이 `n` 이상이면 `right ↓`, 정답 후보 저장
- 처리 불가능이면 `left ↑` 하며 더 많은 시간 탐색
- **left가 right를 넘어설 때**, `answer`가 최소 시간으로 확정됨
