# 괄호 판단 로직 요약 (스택 기반)

## 나의 흐름 정리

- `(` 괄호가 오면 → `push()` 해서 스택에 저장
- 나중에 `)` 닫는 괄호가 오면 → `pop()` 해서  
  가장 최근에 저장된 괄호를 꺼냄
- 꺼낸 게 `"("` 이면 → 짝이 맞으니까 통과
- 꺼낸 게 `"("`이 아니면 → 짝이 안 맞으니까 → `return "no"`
- 👉 즉, **짝이 맞으면 스택에서 제거되어 남지 않음**

---

## 예시로 더 확실하게

```js
line = "( )"
stack = []

1. char = '('
   → stack = ['(']

2. char = ')'
   → stack.pop() = '('
   → '('랑 ')' 짝 맞음 → 그냥 넘어감
   → stack은 이제 []
```

→ 스택이 비었으니 `"yes"` 출력됨

---

## KeyWord Point

- 짝이 맞으면 `push()` 했던 괄호는 `pop()`으로 꺼내면서 **사라지고**,
- 스택엔 **아무것도 남지 않게** 됨.

---

## 요약 문장

> **닫는 괄호가 왔을 때**,  
> **이전에 `push()` 해뒀던 괄호를 `pop()` 해서 꺼내보고**,  
> **짝이 맞으면 제거되고**,  
> **짝이 안 맞으면 바로 `return "no"` 하는 구조**.
